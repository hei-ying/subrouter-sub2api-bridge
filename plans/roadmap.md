# Roadmap

## Phase 1

- capture current in-place adapter behavior
- stabilize login/register/token flows
- document all known incompatibilities

## Phase 2

- implement compatibility matrix for usage/logs/subscriptions
- define payment translation strategy
- hide or downgrade unsupported distributor-only features

## Phase 3

- extract adapter into a standalone bridge layer
- add tests for key workflows
- prepare upstream contribution or deployment documentation
